## Que pouvez-vous faire avec MemFree Open Source

-   Vous pouvez déployer **votre propre moteur de recherche hybride AI** basé sur vos signets, notes et documents
-   Vous pouvez commencer par indexer les meilleures ressources sur un sujet et ensuite construire un **moteur de recherche AI spécifique à un domaine**
-   Vous pouvez d'abord indexer tous les documents d'un site web, puis créer un **assistant de réponse intelligent pour un site web**

## Comment déployer en un clic

<Steps>

### Déployer le backend avec Zeabur

<a href="https://zeabur.com/templates/CE71SC?referralCode=memfree">
    <img src="https://zeabur.com/button.svg" alt="Déployer sur Zeabur" />
</a>

-   Vous pouvez obtenir `UPSTASH_REDIS_REST_URL` et `UPSTASH_REDIS_REST_TOKEN` à partir de [Upstash Redis](https://upstash.com/docs/redis/overall/getstarted)
-   Vous pouvez utiliser `openssl rand -base64 32` pour générer une chaîne aléatoire pour le token API

### Déployer le frontend avec Vercel

<a href="https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fmemfreeme%2Fmemfree&env=UPSTASH_REDIS_REST_URL,UPSTASH_REDIS_REST_TOKEN,OPENAI_API_KEY,SERPER_API_KEY,AUTH_SECRET&envDescription=https%3A%2F%2Fgithub.com%2Fmemfreeme%2Fmemfree%2Fblob%2Fmain%2Ffrontend%2Fenv-example&project-name=memfree&repository-name=memfree&demo-title=MemFree&demo-description=MemFree – Moteur de recherche hybride AI&demo-url=https%3A%2F%2Fwww.memfree.me%2F&demo-image=https%3A%2F%2Fwww.memfree.me%2Fog.png&root-directory=frontend">
    <img src="https://vercel.com/button" alt="Déployer avec Vercel" />
</a>

-   Vous pouvez obtenir `OPENAI_API_KEY` à partir de [OpenAI](https://platform.openai.com)
-   Le MEMFREE_HOST est `https://${PUBLIC_DOMAIN}.zeabur.app`
-   Le AUTH_SECRET est une chaîne aléatoire, vous pouvez utiliser `openssl rand -base64 32` pour le générer
-   Le API_TOKEN doit être le même que celui que vous avez utilisé dans le backend

### Quelques variables optionnelles

-   Pour activer une recherche plus rapide, vous pouvez définir `SERPER_API_KEY` dans Vercel, vous pouvez obtenir `SERPER_API_KEY` à partir de [Serper Dev](https://serper.dev)
-   Pour activer la journalisation AXIOM, vous pouvez définir `AXIOM_TOKEN` dans Vercel et Zeabur, vous pouvez obtenir `AXIOM_TOKEN` à partir de [Axiom](https://axiom.co)
-   Pour activer un parseur de page web plus rapide, vous pouvez définir `JINA_KEY` dans Zeabur, vous pouvez obtenir `JINA_KEY` à partir de [Jina Reader](https://jina.ai/reader)

#### 3.1 Bucket AWS S3 Express

Afin de faciliter le déploiement rapide en un clic pour les développeurs, la base de données vectorielle de memfree stockera les données dans un répertoire local temporaire. **Si vous souhaitez déployer dans un environnement de production, veuillez utiliser un bucket AWS S3 Express**

1. Vous pouvez obtenir des variables AWS. Consultez [Comment créer un bucket S3 Express](https://www.memfree.me/search?q=How%20To%20Create%20S3%20Express%20One%20zone%20Bucket%20and%20get%20ACCESS_KEY,%20SECRET%20KEY%20and%20BUCKET%20name)
2. Vous devez définir les variables suivantes dans le service vectoriel Zeabur :
    - `AWS_ACCESS_KEY_ID`
    - `AWS_SECRET_ACCESS_KEY`
    - `AWS_REGION`
    - `AWS_BUCKET`
